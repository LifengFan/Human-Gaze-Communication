    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><h1><a id="user-content-deformable-convolutional-networks" class="anchor" aria-hidden="true" href="#deformable-convolutional-networks"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Deformable Convolutional Networks</h1>
<h2><a id="user-content-update" class="anchor" aria-hidden="true" href="#update"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Update</h2>
<p><strong>[04/15/2019] The PyTorch version of deformable convolution operators are available in the <a href="https://github.com/open-mmlab/mmdetection">mmdetection codebase</a>. They are very efficient!</strong></p>
<p><strong>[12/01/2018] We updated the deformable convolution operator to be the same as those utilized in the <a href="https://arxiv.org/abs/1811.11168" rel="nofollow">Deformale ConvNets v2</a> paper. A possible issue when the sampling location is outside of image boundary is solved. The issue may cause deteriated performance on ImageNet classification. Note that the current deformable conv layers in both the official MXNet and the PyTorch codebase still have the issue. So if you want to reproduce the results in Deformable ConvNets v2, please utilize the updated layer provided here. The efficiency at large image batch size is also improved. See more details in <a href="https://github.com/msracver/Deformable-ConvNets/blob/master/DCNv2_op/README.md">DCNv2_op/README.md</a>.</strong></p>
<ul>
<li>The full codebase of Deformable ConvNets v2 would be available later. But it should be easy to reproduce the results with the updated operator.</li>
</ul>
<p><strong>[10/2017] We released the training/testing code and pre-trained models of Deformable FPN, which is the foundation of our COCO detection 2017 entry.</strong> Slides at <a href="http://presentations.cocodataset.org/COCO17-Detect-MSRA.pdf" rel="nofollow">COCO 2017 workshop</a>.</p>
<p><strong><a href="https://github.com/bharatsingh430/Deformable-ConvNets">A third-party improvement</a> of Deformable R-FCN + Soft NMS</strong></p>
<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h2>
<p><strong>Deformable ConvNets</strong> is initially described in an <a href="https://arxiv.org/abs/1703.06211" rel="nofollow">ICCV 2017 oral paper</a>. (Slides at <a href="http://www.jifengdai.org/slides/Deformable_Convolutional_Networks_Oral.pdf" rel="nofollow">ICCV 2017 Oral</a>)</p>
<p><strong>R-FCN</strong> is initially described in a <a href="https://arxiv.org/abs/1605.06409" rel="nofollow">NIPS 2016 paper</a>.</p>

<ul>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<ol>
<li>Clone the Deformable ConvNets repository, and we'll call the directory that you cloned Deformable-ConvNets as ${DCN_ROOT}.</li>
</ol>
<pre><code>git clone https://github.com/msracver/Deformable-ConvNets.git
</code></pre>
<ol start="2">
<li>
<p>For Windows users, run <code>cmd .\init.bat</code>. For Linux user, run <code>sh ./init.sh</code>. The scripts will build cython module automatically and create some folders.</p>
</li>
<li>
<p>Install MXNet:</p>
<p><strong>Note: The MXNet's Custom Op cannot execute parallelly using multi-gpus after this <a href="https://github.com/apache/incubator-mxnet/pull/6928">PR</a>. We strongly suggest the user rollback to version <a href="https://github.com/dmlc/mxnet/tree/998378a">MXNet@(commit 998378a)</a> for training (following Section 3.2 - 3.5).</strong></p>
<p><em><strong>Quick start</strong></em></p>
<p>3.1 Install MXNet and all dependencies by</p>
<pre><code>pip install -r requirements.txt
</code></pre>
<p>If there is no other error message, MXNet should be installed successfully.</p>
<p><em><strong>Build from source (alternative way)</strong></em></p>
<p>3.2 Clone MXNet and checkout to <a href="https://github.com/dmlc/mxnet/tree/998378a">MXNet@(commit 998378a)</a> by</p>
<pre><code>git clone --recursive https://github.com/dmlc/mxnet.git
git checkout 998378a
git submodule update
# if it's the first time to checkout, just use: git submodule update --init --recursive
</code></pre>
<p>3.3 Compile MXNet</p>
<pre><code>cd ${MXNET_ROOT}
make -j $(nproc) USE_OPENCV=1 USE_BLAS=openblas USE_CUDA=1 USE_CUDA_PATH=/usr/local/cuda USE_CUDNN=1
</code></pre>
<p>3.4 Install the MXNet Python binding by</p>
<p><em><strong>Note: If you will actively switch between different versions of MXNet, please follow 3.5 instead of 3.4</strong></em></p>
<pre><code>cd python
sudo python setup.py install
</code></pre>
<p>3.5 For advanced users, you may put your Python packge into <code>./external/mxnet/$(YOUR_MXNET_PACKAGE)</code>, and modify <code>MXNET_VERSION</code> in <code>./experiments/rfcn/cfgs/*.yaml</code> to <code>$(YOUR_MXNET_PACKAGE)</code>. Thus you can switch among different versions of MXNet quickly.</p>
</li>
<li>
<p>For Deeplab, we use the argumented VOC 2012 dataset. The argumented annotations are provided by <a href="http://home.bharathh.info/pubs/codes/SBD/download.html" rel="nofollow">SBD</a> dataset. For convenience, we provide the converted PNG annotations and the lists of train/val images, please download them from <a href="https://1drv.ms/u/s!Am-5JzdW2XHzhqMRhVImMI1jRrsxDg" rel="nofollow">OneDrive</a>.</p>
</li>
</ol>

<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<ol>
<li>
<p>All of our experiment settings (GPU #, dataset, etc.) are kept in yaml config files at folder <code>./experiments/rfcn/cfgs</code>, <code>./experiments/faster_rcnn/cfgs</code> and <code>./experiments/deeplab/cfgs/</code>.</p>
</li>
<li>
<p>Eight config files have been provided so far, namely, R-FCN for COCO/VOC, Deformable R-FCN for COCO/VOC, Faster R-CNN(2fc) for COCO/VOC, Deformable Faster R-CNN(2fc) for COCO/VOC, Deeplab for Cityscapes/VOC and Deformable Deeplab for Cityscapes/VOC, respectively. We use 8 and 4 GPUs to train models on COCO and on VOC for R-FCN, respectively. For deeplab, we use 4 GPUs for all experiments.</p>
</li>
<li>
<p>To perform experiments, run the python scripts with the corresponding config file as input. For example, to train and test deformable convnets on COCO with ResNet-v1-101, use the following command</p>
<pre><code>python experiments\rfcn\rfcn_end2end_train_test.py --cfg experiments\rfcn\cfgs\resnet_v1_101_coco_trainval_rfcn_dcn_end2end_ohem.yaml
</code></pre>
<p>A cache folder would be created automatically to save the model and the log under <code>output/rfcn_dcn_coco/</code>.</p>
</li>
<li>
<p>Please find more details in config files and in our code.</p>
</li>
</ol>

<h2><a id="user-content-faq" class="anchor" aria-hidden="true" href="#faq"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>FAQ</h2>
<p>Q: It says <code>AttributeError: 'module' object has no attribute 'DeformableConvolution'</code>.</p>
<p>A: This is because either</p>
<ul>
<li>
<p>you forget to copy the operators to your MXNet folder</p>
</li>
<li>
<p>or you copy to the wrong path</p>
</li>
<li>
<p>or you forget to re-compile</p>
</li>
<li>
<p>or you install the wrong MXNet</p>
<p>Please print <code>mxnet.__path__</code> to make sure you use correct MXNet</p>
</li>
</ul>
<p><br><br>
Q: I encounter <code>segment fault</code> at the beginning.</p>
<p>A: A compatibility issue has been identified between MXNet and opencv-python 3.0+. We suggest that you always <code>import cv2</code> first before <code>import mxnet</code> in the entry script.</p>
<p><br><br>
Q: I find the training speed becomes slower when training for a long time.</p>
<p>A: It has been identified that MXNet on Windows has this problem. So we recommend to run this program on Linux. You could also stop it and resume the training process to regain the training speed if you encounter this problem.</p>
<p><br><br>
Q: Can you share your caffe implementation?</p>
<p>A: Due to several reasons (code is based on a old, internal Caffe, port to public Caffe needs extra work, time limit, etc.). We do not plan to release our Caffe code. Since current MXNet convolution implementation is very similar to Caffe (almost the same), it is easy to port to Caffe by yourself, the core CUDA code could be kept unchanged. Anyone who wish to do it is welcome to make a pull request.</p>
